## 核心设计变更（本次更新）

### 1. Role 可扩展属性
- `properties: dict[str, Any]` 参数 Map，用于领域特定状态

### 2. StateMachine 持有 Action 映射
- `actions: dict[ActionType, Action]`
- `get_available_action_types()` 按当前状态返回可用 ActionType
- `process(action_type, role)` 取出对应 Action 执行

### 3. Action 是一等对象，接收完整 Role
- `beforeExecute(role)` — 执行前由 Role 调用，可做前置检查
- `execute(role)` — 由状态机调用，可访问 Role 全部属性
- `afterExecute(role)` — 执行后由 Role 调用，可通过 `role.relations` 做级联传播

### 4. Role-状态机-Action 调用链
```
外部/Entity -> Role.act(action_type, payload)
                -> Role 设置 payload 到 Action
                -> Role 调用 action.beforeExecute(role)
                -> 状态机调用 action.execute(role)
                -> 状态机更新状态，Role 保存
                -> Role 调用 action.afterExecute(role)
                -> Action 自主决定级联传播
```

### 5. 智能体接口合并
- `_think` + `_action` 合并为 `_agent_action(context, messages, skills...)`
- 可选实现，Action.execute 内部可调用 `role._agent_action(...)`

### 6. Entity 内部传播规则
- Entity 初始化时从配置读取 `propagation_rules`
- 收到内部 Role 消息后，按规则构造 `action_type + payload` 转发给其他 Role
- 规则可配置为固定规则、条件规则或 LLM 驱动

### 7. 级联传播
- 框架层只提供通道
- 具体传播逻辑由 `Action.afterExecute` 的实现决定
- 对内对外对 Role 都是统一的 `act(action_type, payload)` 调用

## 核心决策摘要

| 决策 | 内容 |
|------|------|
| Entity | 聚合根，Role的家长，内部消息中介，按规则转发Action |
| Role | 聚合根，含`properties`，保存状态机状态，统一`act(action_type, payload)`入口 |
| StateMachine | 上层注入，持`actions: dict[ActionType, Action]`，负责状态转移 |
| Action | 一等对象，含`beforeExecute`/`execute`/`afterExecute`，主导级联传播 |
| ActionType | 可扩展枚举，StateMachine的actions的key |
| _agent_action | 可选智能体接口，替代 _think/_action |
| Relation | Role的关联列表，边界由Action/状态机逻辑决定 |
