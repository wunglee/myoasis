## 核心设计变更（本次更新）

### 1. 双聚合根模型
| 聚合根 | 职责 |
|--------|------|
| **Entity** | Role的"家长"，内部Role间的中介 |
| **Role** | 行为执行者，自主管理外部关联关系 |

### 2. 关键变更
- **Entity不是网关**：不负责外部消息转发
- **Role直接交互**：通过Relation关联，无需Entity中介
- **Meeting是ActionType**：不是独立方法，是枚举值
- **Strategy是关键抽象**：Action执行生成有状态的Strategy对象
- **关联约束行为**：无关联=无法交互

### 3. 架构示例
```
Entity A (人)                    Entity B (人)
├─ Role A1 (父亲) ←────关联────→ Role B1 (儿子)
│     
└─ Role A2 (雇员) ←────关联────→ Role C1 (老板) [Entity C]
```

- 父亲Role ↔ 儿子Role：直接交互（同领域，通过Relation）
- 父亲Role ↔ 雇员Role：通过Entity A中介（不同领域）

## 核心决策摘要

| 决策 | 内容 |
|------|------|
| Entity | 聚合根，Role的家长，内部中介，非网关 |
| Role | 聚合根，行为载体，自主管理外部关联 |
| ActionType | 可扩展枚举，Meeting是其中一种 |
| Action | 行为命令，执行后生成Strategy |
| Strategy | 有状态策略对象，核心抽象 |
| Meeting | Strategy子类，多方关联的有状态策略 |
| Relation | Role间关联，决定Action可执行性 |

## 上下文备注

设计强调"绝对抽象"——核心层不应对任何领域（包括社交）有假设。所有现有OASIS社交功能将在Phase 4作为"社交领域插件"实现。

双聚合根模型是关键架构创新：
- Entity = 身份聚合 + 内部中介
- Role = 行为聚合 + 外部关联
- 这种分离允许一个Entity拥有多个领域Role，每个Role自主管理同领域的外部关系
